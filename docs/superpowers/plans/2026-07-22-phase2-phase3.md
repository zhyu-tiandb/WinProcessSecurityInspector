# Phase 2 And Phase 3 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Complete the browser-focused diagnostics and the main full-diagnostics collectors needed before GUI/productization.

**Architecture:** Keep the current header-only `wpsi_core` shape for this stage, adding focused inspectors under `include/wpsi/inspectors/` and small CLI commands in `src/cli/main.cpp`. Data remains read-only, partial failures are represented in `FieldValue`/`Result`, and CLI text/JSON/Markdown output is generated from the same report model.

**Tech Stack:** C++20, Windows API, CMake, CTest, no new third-party dependency.

## Global Constraints

- Target Windows 10 x64 and Windows 11 x64.
- Do not elevate, inject, alter target processes, modify UAC, or change registry/service/task configuration.
- Ordinary user mode must return partial data where possible.
- MSVC runtime stays static via `CMAKE_MSVC_RUNTIME_LIBRARY`.
- Phase 4 GUI and xianc2 wrapper are out of scope for this plan.
